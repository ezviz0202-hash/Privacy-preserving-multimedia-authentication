# PrivAuth-MM

**Privacy-Preserving Multimedia Authentication via Deep Learning**

> A lightweight framework combining privacy-constrained feature extraction, differential-privacy hash authentication, and content-adaptive thresholding — designed for robust forgery detection without sensitive attribute leakage.

---

## Motivation

Modern multimedia forensics faces a dual challenge: authentication systems must be **accurate** (detect forgeries reliably) while being **privacy-preserving** (not leak sensitive personal attributes from feature embeddings). This project proposes and implements an end-to-end framework addressing both concerns, with special attention to practical deployability on CPU-only hardware.

This project is part of ongoing research interest in multimedia security and AI-aided forensics, aligned with the research directions of Prof. Minoru Kuribayashi (Tohoku University, Information Security Lab).

---

## Innovation Points

### 1. Privacy-Suppressed Feature Extractor (`feature_extractor.py`)
A MobileNetV2 backbone augmented with a **Privacy Suppression Layer (PSL)** — a trainable feature mask that, during adversarial fine-tuning, suppresses embedding dimensions correlated with sensitive attributes (gender, age, identity) while preserving authentication-discriminative information.

```
Image → MobileNetV2 features → Projection head → PSL (σ(w) ⊙ f) → L2 norm → Embedding
```

### 2. DP Hash-Based Authentication Protocol (`auth_protocol.py`)
A two-stage verification protocol:
- **Stage 1 (hard)** — HMAC-SHA256 over quantised DP-noised embeddings
- **Stage 2 (soft)** — cosine similarity fallback for content-preserving distortions

Gaussian noise is calibrated via the **analytic Gaussian mechanism**: σ = Δf · √(2 ln(1.25/δ)) / ε, ensuring formal (ε, δ)-DP guarantees.

```
Enrol:  embedding → Gaussian noise (ε, δ) → quantise → HMAC → store template
Verify: embedding → Gaussian noise (ε, δ) → quantise → HMAC → compare + cosine sim
```

### 3. Content-Adaptive Threshold (`adaptive_threshold.py`)
Authentication thresholds are tuned per content type to maintain FPR < 5% across common distortions (JPEG Q≥70, ±5% crop, Gaussian blur σ≤1):

| Content Type | Threshold | Rationale |
|:---:|:---:|:---|
| Face | 0.82 | High structural consistency |
| Document | 0.88 | Near-binary texture, very stable |
| Scene | 0.70 | Rich intra-class variation |
| Unknown | 0.75 | Conservative fallback |

Content type is determined via lightweight image statistics (colour uniformity, skin-tone ratio) combined with ImageNet class predictions — **no additional model needed**.

## Quick Start

```bash
# 1. Clone
git clone https://github.com/<your-username>/PrivAuth-MM.git
cd PrivAuth-MM

# 2. Install dependencies (CPU-only, no GPU required)
pip install -r requirements.txt

# 3. Run demo
python demo.py
```

The demo will:
- Download two Creative Commons sample images
- Enrol them into the template database
- Run 7 attack types (JPEG mild/heavy, crop mild/heavy, blur, noise, brightness)
- Verify all images and print authentication results
- Save a similarity bar chart to `results/similarity_chart.png`

## Privacy–Accuracy Trade-off

The privacy budget ε controls the noise level injected into embeddings:

| ε | Privacy guarantee | Authentication accuracy |
|:---:|:---:|:---:|
| 0.1 | Very strong | Lower (high noise) |
| 1.0 | Strong | Balanced ✓ |
| 5.0 | Moderate | Higher |
| ∞  | None (baseline) | Highest |

Pass `epsilon=` to `PrivAuthPipeline(epsilon=...)` to explore this trade-off.

## Research Context

This project implements the core framework described in my research plan **"Secure and Privacy-Preserving Multimedia Authentication Using Deep Learning Features"**, submitted as part of my application to Tohoku University Graduate School of Information Sciences.

The design is informed by:
- Recent works on federated learning for multimedia forensics
- Differential privacy mechanisms (Dwork et al.)
- Perceptual hashing and robust image fingerprinting
- AI-aided forgery detection (DeepFake forensics)

---

## Limitations & Future Work

- The PSL is currently initialised at identity (ones) and requires adversarial fine-tuning with sensitive-attribute labels to provide formal privacy guarantees against attribute inference.
- Thresholds are hand-calibrated; future work will use held-out calibration sets and isotonic regression.
- Extension to video authentication and federated multi-client training is planned.

## Experiment: Privacy Budget vs. Authentication Accuracy

`experiments/privacy_tradeoff.py` runs the full pipeline across six DP budgets
(ε = 0.1, 0.5, 1.0, 2.0, 5.0, ∞) and measures:

- **TAR** (True Accept Rate): authentic images correctly accepted
- **FAR** (False Accept Rate): attacked images wrongly accepted

```bash
python experiments/privacy_tradeoff.py
```

Outputs `results/privacy_tradeoff.png` and `results/tradeoff_data.csv`.

The left chart shows TAR/FAR per epsilon. The right chart shows the
privacy-accuracy tradeoff curve — smaller ε = stronger privacy but lower TAR.
The sweet spot in this demo is around **ε = 1.0**: TAR stays high while FAR
remains low, consistent with the analytic Gaussian mechanism's theoretical
guarantees.

This experiment directly supports Innovation Point 2 (DP hash authentication)
by providing empirical evidence of the privacy-accuracy tradeoff.
